JavaScript 数据类型和数据结构

March 07 2018

动态性

JavaScript 是一种弱类型或动态语言，变量不需要先声明类型，程序在运行时确定变量的类型。所以，一个变量可保存不同类型的数据

原始类型

  • Boolean
  • String
  • Number
  • null
  • undefined
  • Object
  • Symbol (ECMAScript 6 新定义)

也可按照值类型和引用类型分类 值类型：五种基本类型 引用类型：数组，函数，对象

Number

NaN

The global NaN property is a value representing Not-A-Number

NaN 是 number 类型的一个值，但是 NaN 值与任何 number 类型的值比较都不相等，与任何 number 类型的值比较都返回 false，通常某些数学函数例如 Math.sqrt(-1) 计算不出结果的会返回 NaN

有两个方法需要注意

  • isNaN() 方法是将值转换成 number 时，失败时返回 true
  • Number.isNaN() 方法为当前值是 NaN 时才返回 true

Infinity

The global Infinity property is a numeric value representing infinity

Infinity 表示 number 类型中的正无穷大，它等于自身，比其他 number 类型的值都要大

Number 中的方法

Number

ES6 中的数值常用扩展

  • 二进制和八进制

用前缀 0b 和 0o 分别表示 二进制和八进制

  • Number.isFinite() 和 Number.isNaN()

Number.isFinite() 判断数值是否为有限的 Number.isNaN() 判断数值是否为 NaN 这个两个函数区别于全局的 isFinite() 和 isNaN() 在于，全局的这两个函数是先对参数做 Number 类型转换后再判断的，而 Number.isFinite() 对于非数值一律返回 false， Number.isNaN() 只有对于 NaN 才返回 true，非 NaN 一律返回 false。

  • parseInt() 与 parseFloat() 移植到 Number

  • Number.isInteger()

此函数用于判断是否为整数，对于非数值，返回 false 但是要注意的是 JavaScript 采用 IEEE 754 标准，数值存储为64位双精度格式，数值精度最多可以达到 53 个二进制位（1 个隐藏位与 52 个有效位）。如果数值的精度超过这个限度，第54位及后面的位就会被丢弃，这种情况下，Number.isInteger可能会误判

String

JavaScript 内部，字符以 UTF-16 的格式储存，每个字符固定为2个字节。对于那些需要4个字节储存的字符（Unicode 码点大于0xFFFF的字符），JavaScript 会认为它们是两个字符。

ES6 中的字符串常用扩展

  • includes()，startsWith()，endsWith()

includes() 判断是否包含参数字符串 startsWith() 判断是否以参数字符串开头 endsWith() 判断是否以参数字符串结尾

  • padStart()，padEnd()

  • 模板字符串

Boolean

Do not confuse the primitive Boolean values true and false with the true and false values of the Boolean object.

不要混淆原始类型中的 boolean 与 Boolean 类对象

Symbol

ES6 引入了一种新的原始数据类型Symbol，表示独一无二的值，Symbol 值通过Symbol函数生成，但前面不能加 new 关键字，因为它不是对象，它是一种类似字符串的数据类型

  • 因为 Symbol 独一无二，所以可以作为对象的属性属性名

  • 作为独一无二的常量

  • 消除魔术字符串

魔术字符串指的是在代码中出现，与代码形成“强耦合”的字符串

当要对魔术字符串进行更新时，要修改多处，非常不利于代码的维护，所以应该将魔术字符串用变量代替，抽取出来

typeof

typeof 函数返回变量的原始类型或 object 或 function

undefined 与 null

  • undefined undefined 分配给已经声明但是没有实际值的变量
  • null null 表示通常故意指向不存在或无效的对象或地址的引用

null 表示"没有对象"，即该处不应该有值。典型用法是

  • 作为函数的参数，表示该函数的参数不是对象
  • 作为对象原型链的终点

undefined 表示"缺少值"，就是此处应该有一个值，但是还没有定义。

  • 变量被声明了，但没有赋值时，就等于undefined
  • 调用函数时，应该提供的参数没有提供，该参数等于 undefined
  • 对象没有赋值的属性，该属性的值为 undefined
  • 函数没有返回值时，默认返回 undefined
